Conclusion

Concepts pratiqués :

- Structures de contrôle : boucles, conditions.

- Organisation du code : fonctions.

- Gestion des entrées utilisateur.

Apprentissages :

- Listes pour stocker lettres proposées.

- Code réutilisable grâce aux fonctions.

- Validation stricte des entrées utilisateur.

- Gestion des erreurs et essais (conditions).

- Affichage dynamique (underscore pour lettres manquantes).

Difficultés :

- Vérification d'entrées valides (lettres uniquement).

- Prévention des doublons (lettres déjà utilisées).

- Gestion des accents et des majuscules/minuscules.

- Logique de fin de jeu (victoire/défaite).

Appréciations :

- Personnalisation : niveaux de difficulté, erreurs possibles.

- Satisfaction : projet fonctionnel, bugs résolus.

- Collaboration enrichissante.
